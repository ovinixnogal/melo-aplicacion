Se ha eliminado el rehash basado en la comprobación antigua.
Ahora se utiliza 

needs_rehash
 justo después de una verificación de contraseña exitosa.
Si el hash necesita actualización, se rehashea la contraseña (que ya está en memoria tras el login) y se guarda en la base de datos de forma transparente para el usuario.
Seguridad forzada: Los usuarios con contraseñas en texto plano (legacy) ahora fallarán el login y deberán utilizar el flujo de "Olvidé mi contraseña" para regenerar su acceso con un hash seguro, protegiendo así la base de datos de compromisos futuros.
He verificado que la lógica mantiene la funcionalidad para los usuarios activos y es retrocompatible con el sistema de sesiones existente.

He solucionado la **Vulnerabilidad 4: Rate Limiter Persistente con Redis**.

### Cambios realizados:

1.  **Dependencia de Redis**: He añadido `redis>=5.0.0` a [requirements.txt](cci:7://file:///c:/Users/Nixon/Desktop/Apps/melo-aplicacion/requirements.txt:0:0-0:0).
2.  **Sistema de Rate Limiting Inteligente**:
    *   **Backend de Redis**: He implementado un cliente Redis que se activa automáticamente si detecta la variable de entorno `REDIS_URL` (proporcionada por el plugin de Redis en Railway).
    *   **Estrategia de Ventana Deslizante (Sliding Window)**: Utilizo estructuras `Sorted Set` de Redis para contar los intentos exactos en los últimos 60 segundos por IP. Esto es mucho más preciso que un contador fijo y persiste entre reinicios del servidor.
    *   **Persistencia entre Workers**: Al usar Redis, todos los procesos del servidor (gunicorn/uvicorn workers) comparten el mismo contador de intentos, evitando que un atacante multiplique sus oportunidades.
    *   **Fallback Seguro**: Si Redis no está disponible o falla en tiempo de ejecución, el sistema cae automáticamente a un diccionario en memoria (con auto-limpieza para evitar fugas) para no interrumpir el servicio.
    *   **Protección Anti-DoS**: El límite sigue basándose en la IP para prevenir ataques de denegación de servicio a cuentas de usuario específicas.

### Siguiente paso en Railway:
Asegúrate de agregar el plugin de **Redis** a tu proyecto. Railway inyectará la variable `REDIS_URL` necesaria automáticamente.
